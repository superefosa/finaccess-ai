Backend routes

These are API routes. Most are best used from Swagger UI.

Live backend base

https://finaccess-backend.onrender.com

Main backend endpoints

/                 → No homepage, usually Not Found
/docs             → Swagger UI
/openapi.json     → Raw API schema
/health           → Health check

Auth

/auth/register    → Register user
/auth/login       → Login user
/auth/me          → Get current logged-in user

Profiles

/profiles         → Create or update financial profile
/profiles/me      → Get current user's profile

Applications

/applications             → Create application
/applications/me          → Get current user's applications
/applications/{id}        → Get one application by ID

Admin

/admin/applications                 → List all applications
/admin/applications/{id}/decision   → Approve or reject application
/admin/audit-logs                   → View audit logs

## ▶️ How to Start the App on a Local Machine

There are two ways to run the project locally:

### Option 1: Run with Docker

This starts the full stack together.

#### 1. Train the AI model
```bash
cd ai
python train_model.py

2. Start the application

docker compose up --build

3. Seed demo accounts
Open a new terminal and run:

docker compose exec backend python seed_demo.py

4. Open the app
      •     Frontend: http://localhost:5173
      •     Backend API: http://localhost:8000
      •     API Docs: http://localhost:8000/docs

Demo credentials
      •     Admin: admin@example.com / AdminPass123!
      •     User: user@example.com / UserPass123!

⸻

Option 2: Run without Docker

Use this if running services manually.

Backend
Open Terminal 1:

cd backend
pip install -r requirements.txt
alembic upgrade head
python seed_demo.py
uvicorn app.main:app --reload

Frontend
Open Terminal 2:

cd frontend
npm install
npm run dev

Open the app
      •     Frontend: http://localhost:5173
      •     Backend API: http://localhost:8000
      •     API Docs: http://localhost:8000/docs

Demo credentials
      •     Admin: admin@example.com / AdminPass123!
      •     User: user@example.com / UserPass123!
